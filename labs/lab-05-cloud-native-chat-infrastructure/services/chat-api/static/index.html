Lab 05: Cloud-Native Chat Infrastructure

Ingress and processing are decoupled. The API node accepts chat quickly, Redis buffers workloads, workers persist to PostgreSQL, and archive files are mirrored into MinIO for cold storage.

Why this lab exists
A monolith with in-memory queues is useful but constrained. This lab externalizes the queue and processing path so API responsiveness is less coupled to storage pressure.
Hot path vs cold path
Hot path: WebSocket to API enqueue. Cold path: worker persistence and archive sync. This split helps stabilize user-facing latency while still keeping durable history and retention flow.
Services in play
Chat API, Redis queue and event bus, PostgreSQL durable store, worker processor, MinIO object storage, and Prometheus metrics.
Observe in real time
Use the burst buttons and watch queue depth. You should see messages continue flowing while the worker drains backlog and archive files are synced into object storage.
