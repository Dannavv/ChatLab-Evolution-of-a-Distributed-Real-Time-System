Lab 03: Redis Pub/Sub

Collapse the theory. Use the live example on the right to watch messages move across nodes.

What Problem Redis Solves
Each chat server only knows about its own WebSocket clients. Redis Pub/Sub bridges the gap so every node can relay the same message stream.
Message Flow
A node writes the message to PostgreSQL, publishes it to chat_messages, then every server broadcasts that payload to its local clients.
Why Two Nodes
Open both exposed nodes to see the same message arrive in both browsers, even when you send from only one of them.
What The Metrics Mean
chat_active_connections is local per node, while chat_messages_total and latency reflect the message path on that node.

Live Example

Open http://localhost:8082 and http://localhost:8083 in separate tabs, then send a message from one node and watch it appear on both.
