Lab 07: Real-Time Presence and Delivery

This lab introduces live presence, typing indicators, read receipts, and delivery events. Two nodes synchronize event streams through Redis to keep user state consistent across node boundaries.

What changed from Lab 06
Lab 06 focused on resilience controls. Lab 07 focuses on user-facing real-time semantics: who is online, who is typing, and whether messages were delivered and read.
Event types in this lab
message, presence, typing, read, and delivery. Each event carries user, room, node, timestamp, and event identifiers so clients can render consistent state in real time.
Cross-node consistency
Each node publishes events to Redis and subscribes to the same channel. This allows clients connected to different nodes to observe one shared room state.
How to test it
Open this page on both node ports. Use different users in the same room. Start typing, send messages, and click mark read to verify the event fan-out and status updates.
